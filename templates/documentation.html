Introduction

The Roblox API Hub provides a comprehensive set of endpoints for interacting with the Roblox platform. This API allows you to access information about users, games, groups, friends, assets, and more.

Base URL

http://localhost:5000

Authentication

Currently, this API doesn't require authentication as it's primarily for retrieving public information from Roblox. For endpoints that might eventually require authentication, this will be updated in the future.

Response Format

All API responses are returned in JSON format with a consistent structure:

{ "success": true, // Boolean indicating if the request was successful "data": { ... } // The actual response data } // Error response: { "success": false, "error": { "code": 404, // HTTP status code "message": "User not found" // Error message } }

Error Handling

The API uses standard HTTP status codes to indicate the success or failure of a request. In addition, error responses include a detailed message explaining what went wrong.

Common Error Codes

Status Code Description
400 Bad Request - Invalid input parameters
404 Not Found - The requested resource doesn't exist
429 Too Many Requests - Rate limit exceeded
500 Internal Server Error - Something went wrong on the server

Error Response Example

{ "success": false, "error": { "code": 404, "message": "User not found with ID 123456789" } }

Rate Limiting

To ensure stable performance and to comply with Roblox's own rate limits, this API implements rate limiting. The current limit is 60 requests per minute per client.

Rate Limit Response

When you exceed the rate limit, you'll receive a 429 Too Many Requests response:

{ "success": false, "error": { "code": 429, "message": "Rate limit reached. Try again in 30 seconds" } }
The API handles Roblox's rate limits internally, but you should still implement your own rate limiting to ensure your application doesn't make too many requests at once.

User Endpoints

These endpoints provide access to Roblox user information and allow you to search for users.

GET /api/users/{user_id}

Returns information about a Roblox user by their ID.

Path Parameters
Parameter Type Description
user_id integer The Roblox user ID
Example Response
{ "success": true, "data": { "id": 1234567, "name": "Username", "displayName": "Display Name", "description": "User's description...", "created": "2012-01-01T00:00:00Z", "isBanned": false } }
POST /api/users

Returns information about multiple Roblox users by their IDs.

Request Body
{ "userIds": [1234567, 7654321, 9876543] }
Example Response
{ "success": true, "data": { "data": [ { "id": 1234567, "name": "Username1", "displayName": "Display Name 1" }, { "id": 7654321, "name": "Username2", "displayName": "Display Name 2" }, { "id": 9876543, "name": "Username3", "displayName": "Display Name 3" } ] } }
GET /api/users/search

Searches for Roblox users by keyword. This endpoint always connects to the real Roblox API with enhanced error handling and retry mechanisms.

Query Parameters
Parameter Type Description
keyword string The search term (required). Special characters are automatically sanitized.
limit integer Maximum number of results to return (default: 10, max: 100)
Example Request
GET /api/users/search?keyword=builderman&limit=5
Example Response
{ "success": true, "data": { "data": [ { "id": 156, "name": "Builderman", "displayName": "Builderman", "previousUsernames": [] }, { "id": 1234567, "name": "BuildermanFan123", "displayName": "Builder", "previousUsernames": ["OldName1"] } ] } }

Game Endpoints

GET /api/games/{game_id}

Returns information about a Roblox game by its ID.

Path Parameters
Parameter Type Description
game_id integer The Roblox game ID
Example Response
{ "success": true, "data": { "id": 1234567890, "name": "Game Name", "description": "Game description...", "creator": { "id": 1234567, "type": "User", "name": "Creator" }, "created": "2019-01-01T00:00:00Z", "updated": "2023-01-01T00:00:00Z", "playing": 1000, "visits": 1000000, "maxPlayers": 30, "genre": "Adventure" } }
GET /api/games

Returns games created by a specific user.

Query Parameters
Parameter Type Description
user_id integer The Roblox user ID (required)
limit integer Maximum number of results to return (default: 50)
Example Request
GET /api/games?user_id=1234567&limit=10
Example Response
{ "success": true, "data": { "games": [ { "id": 1234567890, "name": "Game 1", "playing": 100, "visits": 100000 }, { "id": 9876543210, "name": "Game 2", "playing": 50, "visits": 50000 } ] } }
GET /api/games/{game_id}/details

Returns detailed information about a game, including social links and game passes.

Path Parameters
Parameter Type Description
game_id integer The Roblox game ID
Example Response
{ "success": true, "data": { "details": { "id": 1234567890, "name": "Game Name", "description": "Game description...", "creator": { "id": 1234567, "type": "User", "name": "Creator" }, "playing": 1000, "visits": 1000000 }, "socialLinks": { "links": [ { "title": "Website", "url": "https://example.com" } ] }, "gamePasses": { "data": [ { "id": 11111, "name": "VIP Pass", "price": 400 }, { "id": 22222, "name": "Double XP", "price": 200 } ] } } }

Avatar Endpoints

These endpoints provide access to Roblox avatar information and outfit management.

GET /api/users/{user_id}/avatar

Returns information about a user's avatar, including their equipped items, body colors, and scales.

Path Parameters
Parameter Type Description
user_id integer The Roblox user ID

{ "success": true, "data": { "scales": { "height": 1.0, "width": 1.0, "head": 1.0, "depth": 1.0, "proportion": 0.0, "bodyType": 0.0 }, "playerAvatarType": "R15", "bodyColors": { "headColorId": 24, "torsoColorId": 1003, "rightArmColorId": 24, "leftArmColorId": 24, "rightLegColorId": 1001, "leftLegColorId": 1001 }, "assets": [ { "id": 5785985, "name": "Rusty Tetramino of Competence", "assetType": { "id": 8, "name": "Hat" } } ], "defaultShirtApplied": false, "defaultPantsApplied": false, "emotes": [] } }

Path Parameters
Parameter Type Description

Path Parameters
Parameter Type Description

Group Endpoints

GET /api/groups/{group_id}

Returns information about a Roblox group by its ID.

Path Parameters
Parameter Type Description
group_id integer The Roblox group ID
Query Parameters
Parameter Type Description
user_id integer Optional user ID to include user's groups
Example Response
{ "success": true, "data": { "id": 12345, "name": "Group Name", "description": "Group description...", "owner": { "id": 1234567, "type": "User", "name": "OwnerUsername" }, "memberCount": 5000, "isVerified": true, "created": "2015-01-01T00:00:00Z" } }
GET /api/groups/{group_id}/members

Returns members of a Roblox group.

Path Parameters
Parameter Type Description
group_id integer The Roblox group ID
Query Parameters
Parameter Type Description
limit integer Maximum number of results to return (default: 10)
cursor string Pagination cursor
sort_order string Sort order (Asc/Desc)
role_id integer Filter by role ID
Example Response
{ "success": true, "data": { "data": [ { "user": { "id": 1234567, "name": "Member1" }, "role": { "id": 123, "name": "Admin", "rank": 254 } }, { "user": { "id": 7654321, "name": "Member2" }, "role": { "id": 124, "name": "Member", "rank": 1 } } ], "nextPageCursor": "nextPageToken" } }
GET /api/groups/{group_id}/roles

Returns roles in a Roblox group.

Path Parameters
Parameter Type Description
group_id integer The Roblox group ID
Example Response
{ "success": true, "data": { "roles": [ { "id": 123, "name": "Owner", "rank": 255, "memberCount": 1 }, { "id": 124, "name": "Admin", "rank": 254, "memberCount": 5 }, { "id": 125, "name": "Member", "rank": 1, "memberCount": 4994 } ] } }
